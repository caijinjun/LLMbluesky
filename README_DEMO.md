# ATC HMI BlueSky Demo Package

This package contains a runnable BlueSky visual demo, a dynamic sector scenario, a safety-constrained resolution module, and an LLM-style explanation wrapper.

## Quick Start

### Stable GUI demo

```powershell
.\RUN_BLUESKY_GUI.ps1
```

In BlueSky:

1. Open the bottom `AI` tab.
2. Click `Reset sector`.
3. Select `Preference`: `speed_first` or `altitude_first`.
4. Keep `LLM wrapper` as `template_explainer`.
5. Click `Start auto traffic`.

The GUI will spawn aircraft, detect predicted conflicts, issue verified BlueSky commands, and show standard instructions plus a short explanation.

The visual panel reads live BlueSky aircraft data when available. If the QtGL data stream is delayed or sparse during a short demo, it falls back to route metadata generated by the AI panel so the conflict-detection, decision, and LLM-explanation workflow remains visible.

### GUI demo with mock LLM API

Open one PowerShell window:

```powershell
.\RUN_MOCK_LLM.ps1
```

Open a second PowerShell window:

```powershell
.\RUN_BLUESKY_GUI_WITH_MOCK_LLM.ps1
```

In the `AI` tab, set `LLM wrapper` to `openai_compatible_api`. The GUI will call the local mock endpoint and display the returned explanation. The issued commands still come from the local safety verifier.

### Headless smoke test

```powershell
.\RUN_HEADLESS_SMOKE.ps1
```

Expected result: `success: true`, `num_loss_events: 0`, and `fallback_calls: 0` for the smoke run.

## Package Layout

- `bluesky_project/`: runnable BlueSky project with the AI/HMI panel.
- `headless_validation/`: detached validation scripts and previous stress-test result.
- `llm/`: local OpenAI-compatible mock server for demo.
- `docs/LLM_VISUAL_DEMO_DESIGN.md`: design explanation for the visual HMI and LLM wrapper.

## Design Boundary

The LLM layer is an explanation and instruction-generation wrapper. It does not bypass safety verification. The local solver first searches verified altitude and speed actions; the wrapper then converts verified actions into structured fields, standard phraseology, and a rationale.

For final safety evidence, use the headless validation result under `headless_validation/results/`. The GUI fallback is for demonstration continuity, not a replacement for the headless verifier.
